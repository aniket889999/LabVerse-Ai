# LabVerse AI: System Architecture

## Overview
LabVerse AI is a production-grade SaaS platform designed to transform university laboratory management and visitor engagement. The architecture ensures high availability, secure data isolation, and scalable performance to handle multiple user roles.

## Technology Stack

### Frontend Layer
* **Framework:** Next.js (React)
* **Language:** TypeScript
* **Styling:** Tailwind CSS
* **State Management:** Zustand (or Redux Toolkit)
* **Routing:** Next.js App Router
* **Justification:** Next.js provides excellent SEO for the Lab Explorer and fast rendering (SSR/SSG) for dynamic lab data. TypeScript ensures type safety across the monorepo.

### Backend API Layer
* **Framework:** Node.js with Express (or NestJS)
* **Language:** TypeScript
* **Validation:** Zod
* **Authentication:** JWT (JSON Web Tokens) with stateless verification.
* **Justification:** A unified JavaScript/TypeScript stack allows sharing types between the frontend and backend. Express provides a lightweight, highly customizable API layer.

### Data Layer
* **Primary Database:** PostgreSQL
* **ORM:** Prisma
* **Caching Layer:** Redis (for session management, rate limiting, and caching frequently requested lab data).
* **File Storage:** AWS S3 (for project repository files, images, and tour media).

### AI & Machine Learning Layer
* **AI Lab Guide Engine:** Integration with an LLM provider (e.g., OpenAI API or an internally hosted LLaMA model) using LangChain.js to provide conversational interfaces for the Lab Guide.

## Deployment Architecture

### Cloud Provider (Recommended: AWS or Vercel + AWS)
1.  **Frontend:** Deployed on **Vercel** for global Edge CDN distribution, automated CI/CD, and optimized image/asset serving.
2.  **Backend Services:** Docker containers orchestrated via **AWS ECS (Fargate)** for serverless compute, placed behind an Application Load Balancer.
3.  **Database:** **AWS RDS for PostgreSQL** configured for multi-AZ high availability.
4.  **Cache:** **AWS ElastiCache for Redis**.

## Module Integration

*   **Lab Explorer:** Frontend requests cached lab data from Redis/PostgreSQL via the Backend API.
*   **AI Lab Guide:** Frontend maintains a WebSocket or streaming HTTP connection to the Backend, which proxies requests to the AI engine to maintain secure API keys.
*   **Project Repository:** Stores metadata in PostgreSQL and raw files in S3. Pre-signed URLs are generated by the Backend for secure client-side uploads/downloads.
*   **Visit Booking:** Synchronous REST API flow updating PostgreSQL state with notification triggers.
*   **Analytics Dashboard:** Background cron jobs aggregate raw event data into materialized views in PostgreSQL for fast frontend querying.
